7. Create a for loop function to explore Ks from 2 to 10 for each method using the Silhouette score and display the results with a plot.

explore <- function(data, pretty_name) {
  avg_sil_widths <- vector(mode="numeric", length=9)
  for (i in seq_along(avg_sil_widths)) {
    # Returns a k x 3 matrix with attributes.
    k   <- i + 1
    sil <- silhouette(cutree(data, k), md.dists)
    avg_sil_width <- mean(sil[, 3])
    print(avg_sil_width)
    avg_sil_widths[i] <- avg_sil_width
  }
  plot(2:10, avg_sil_widths, type="b", # Render both points and lines.
       main=paste("Silhouette method (", pretty_name, ")", sep=""),
       xlab="Cluster count (k)", ylab="Average Silhouette width",
       cex=0.5)
}

B. K-means vs. Hierarchical

1. Revisit your K-means outputs from the previous activity and compare how the clusters were determined.

In Activity 07 I determined that the optimal number of clusters was 3 using the Elbow method. However in this activity I found that they were actually 4 and 5 for the Single Linkage and Ward’s Methods, respectively,

2. Are the clusters consistent?

If not, what do you think causes this, and how might it influence your interpretation of biological patterns in the data?

The clusters are not consistent. I think that this has to do with the algorithms we used. In Activity 07 we basically brute forced it and only checked up to k=10. For this activity we instead used functions that are (probably) more robust and statistically sound. The hclust() function must have been really well tested.

When it comes to interpreting biological patterns, even a difference of 1 in the number of clusters (k) like 3 vs. 4 is significant. Clusters that are grouped together when k=3 could be separate when k=4, for example. So things that you consider “related” in K-means could be considered not related in Hierachical Clustering. Another useful thing that HC provides is the ordering which you could use to determine the hierarchy, hence its name. It reminds me of a phlogenetic tree (like from Systematics) which is really useful to determine how related any one point is from the others.
